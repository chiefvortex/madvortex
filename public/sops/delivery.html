Workflow

Figure 1 — Delivery Workflow
Figure 2 — Approval Chain

Follow these steps in order. No shortcuts.

01
Artist
Export final renders with correct naming convention
Use the exact naming format from Section 06. If unsure, ask before exporting.
Same day as completion
02
Pranay Bubna
Review all renders for technical quality
Check resolution, frame rate, codec, color space, audio. Flag and send back if anything fails.
Within 24 hours
03
Naveen Kumar
Final review and sign-off
Does it match the brief? Is quality acceptable? If yes, sign off. Nothing leaves without this.
Within 24 hours of Pranay approval
04
Naveen Kumar
Upload approved files to Frame.io
Create project if needed. Set to "awaiting feedback" mode.
Same day as sign-off
05
Client
Review and provide feedback on Frame.io
Only approved channel for delivery feedback.
Per project contract
06
Naveen Kumar
Archive final approved files
Move to Work of Vortex on Google Drive. Keep working files in project folder.
Same day as client sign-off

Failure Cases & Escalation

Low
Minor naming error. Fix and re-upload.
Medium
QC fails. Send back to artist. Fix in 24h.
High
Client sees unreviewed work. Slack Naveen now.
Critical
Wrong file sent to client. Call Naveen directly.
FailureActionContactTimeframe
Wrong naming conventionRename and re-uploadArtistSame day
QC finds technical issuesSend back with notes on SlackPranay24 hours
Client gives feedback outside Frame.ioRedirect to Frame.io, log the feedback thereNaveenImmediate
Client requests changes after sign-offNew scope. New contract required.NaveenBefore any work starts
Wrong file sent to clientRemove from Frame.io immediately. Replace.Naveen (call)Within 1 hour
Changes requested after final client sign-off = new scope = new contract. No free rework.
