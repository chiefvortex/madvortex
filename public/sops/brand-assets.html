03

Output Standards

Core Rule: Clarity over decoration. Fewer elements create better hierarchy. Every design should feel premium, controlled, intentional, and scalable.

Visual Quality: Clean, aligned, intentional. No random elements. No unfinished work. Every choice has a reason.

Design Rules: Clarity beats decoration every time. Fewer elements create stronger hierarchy. Consistency across all screens and assets is non-negotiable.

File Standards: Proper naming (no "final_v3.psd"). Grouped layers. Organized artboards. Export-ready. Structured so the next person can pick it up immediately.

Clean, aligned visual composition
Clarity prioritized over decoration
Consistent typography and spacing
Grouped layers and organized artboards
Uploaded with correct task link
No missing assets or broken files
04

QC Checklist

Before submission, Designer must verify. If any fail, fix before uploading. No exceptions.

Alignment: No visual imbalance. Every element sits cleanly relative to grid and edges.
Spacing: Consistent gap logic. Margins and padding follow a clear system.
Typography: Font weights, sizes, line-heights are correct and consistent.
Color: No off-brand colors. All colors match locked palette.
Reference Alignment: Design matches approved direction from lock step.
Finish Level: Polished. No rough edges. Ready for client eyes.
QC is not optional. If you skip it, clients see mistakes. Fix before submission, always.
05

Feedback Handling

Feedback is direction, not criticism. Understand intent first. Don't defend. Fix precisely.

Figure 2 — Three-Draft Revision Logic

When feedback arrives:

  1. Understand intent first. What is the person really asking for? Ask clarifying questions if needed.
  2. Don't defend. Your job is to make it better, not explain why it's good as-is.
  3. Fix precisely. Address the feedback directly. Don't over-engineer.

Three-Draft Logic:

  • First draft: Direction and concept. Gets the idea across.
  • Second draft: Refinement and tuning. Tighten based on feedback.
  • Third draft: Finalization. Polish and lock for delivery.

Daily Rhythm

Structure your day around focus blocks, shared progress, and feedback loops within the same day when possible.

1
10 AM Start
Working by 10 AM. Establish focus. Check Slack for blockers.
2
Standup Report
Report active tasks, blockers, and priorities. 2-3 minutes. Slack or in-person.
3
Share Progress Early
Show rough drafts or direction before finishing. No silent work cycles. Feedback loops fast.
4
Iteration Loops
Get feedback, refine, re-share. Same-day when possible. Build momentum.
5
EOD Status Update
Update task status. Document blockers. Set priority for tomorrow.

Failure Modes

Five common failure modes and how to prevent them. Know these. Avoid them.

Failure Mode Impact Prevention
Misaligned / Messy Design Low quality output. Client sees sloppiness. Mandatory QC checklist before submission.
Over-Design / Clutter Loss of clarity. Message buried under decoration. Enforce minimal design rule. Fewer elements, better hierarchy.
Inconsistent Systems Brand breaks across assets. Unprofessional feel. Follow locked direction strictly. Lock once, build consistent.
Slow Iteration Project delays. Momentum lost. Client waiting. Early drafts + fast feedback loops. Share progress daily.
Poor File Structure Team inefficiency. Next person can't pick it up. File hygiene enforced. Clean layers, clear naming, organized artboards.
Final Standard: Your work should feel premium, controlled, intentional, and scalable. Don't decorate. Define.
